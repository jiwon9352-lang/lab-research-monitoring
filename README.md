# Lab Research Monitoring

대학원 랩실 연구동향 대시보드 (PoC)

특정 학교·학과의 연구실(랩) 단위로 연구 동향, 협업 네트워크, 주제 변화(피봇)를 한 화면에서 모니터링하는 웹 대시보드입니다. 별도 서버나 API 키 없이, 브라우저에서 [OpenAlex](https://openalex.org) 학술 데이터 API를 실시간 호출해 동작합니다.

**라이브 데모:** https://jiwon9352-lang.github.io/lab-research-monitoring/

현재 PoC는 서울대학교 환경대학원 도시계획학과 / 사회혁신 분야를 테스트 대상으로 구성되어 있습니다.

---

## 주요 기능

- **연구자 매칭** — 교수(연구자) 이름을 OpenAlex 프로필에 자동 매칭하고, 소속·논문수·h-index로 매칭 정확도를 직접 검증할 수 있습니다.
- **연도별 연구 생산성** — 최근 N년(5/10/15년) 논문 수를 토픽 field별 누적 막대로 시각화합니다.
- **과거 vs 현재 주제 피봇** — 분석 기간을 전·후반으로 나눠 연구 주제 비중 변화(Δ%p)를 정량 비교하여, 연구실의 주제 전환을 한눈에 파악합니다.
- **공저 네트워크** — 연구자–공동저자 관계를 그래프로 표현하고, 여러 교수와 동시에 협업하는 허브 연구자를 강조합니다.
- **랩(PI)별 논문 리스트** — 교수 단위로 최신순 정렬하며, 제목 클릭 시 DOI·원문 페이지로 이동합니다.
- **주제 카테고리별 논문** — 전체 논문을 토픽 field 기준으로 자동 분류해 묶어봅니다.

---

## 데이터 출처

모든 데이터는 [OpenAlex](https://docs.openalex.org/) REST API에서 실시간으로 가져옵니다.

- 2.5억+ 논문, 9천만+ 저자, 기관·토픽·인용 데이터 제공
- API 키 불필요, 무료, CORS 지원으로 정적 페이지에서 직접 호출 가능
- 공저(co-authorship)·토픽(field) 데이터를 구조화 필드로 제공

---

## 동작 방식

1. 학과 탭을 선택하면 해당 분야 교수 시드 명단이 로드됩니다(편집 가능).
2. 각 이름을 OpenAlex 저자로 매칭합니다.
3. 매칭된 저자의 최근 N년 논문을 수집합니다.
4. 수집한 논문에서 토픽·공저·연도 정보를 집계해 차트·네트워크·리스트를 렌더링합니다.

전 과정이 사용자 브라우저 안에서 수행되며, 백엔드 서버가 없습니다.

---

## 한계 (정직성 고지)

- **학과 단위 필터 부재:** OpenAlex는 기관(institution)·저자 단위만 제공하고 학과(department) 단위 필터가 없습니다. 따라서 학과는 교수 시드 명단으로 구성합니다.
- **동명이인 가능성:** 이름 기반 매칭은 동명이인 위험이 있어, 매칭 테이블의 소속·논문수로 직접 확인이 필요합니다.
- **국내 논문 색인 누락:** 한국어권·비색인 자료 일부는 OpenAlex에 없을 수 있습니다.
- 토픽 분류는 OpenAlex의 자동 topic/field 태깅 결과를 사용합니다.

## 실행 / 배포

- **바로 사용:** 위 라이브 데모 링크 접속.
- **로컬 실행:** `index.html` 파일을 브라우저로 열기만 하면 됩니다(설치·빌드 불필요).
- **배포:** GitHub Pages(Settings → Pages → Branch: `main` / root)로 무료 호스팅.

### 다른 학교·학과에 적용하기

`index.html` 상단의 `SEEDS` 객체에서 교수 이름 목록을 바꾸면 됩니다(영문명 권장, 매칭 정확도 향상):

```js
const SEEDS = {
  urban:  ["Young-Sung Lee", "Kyung-Min Kim", "Saehoon Kim", "Soo-eun Chang"],
  social: ["In-Kwon Park", "Kyung-Min Kim"]
};
```
